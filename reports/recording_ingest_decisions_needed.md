# Recording Ingest Decisions Needed

These are the user/recording-team decisions needed before real recording or before cutting the first pilot batch.

1. Recording mode: one continuous/batch-level take or 71 separate files?
2. Should every task begin with a spoken or tapped identifier?
3. Should spoken identifiers remain in raw files or be removed only in normalized copies?
4. Should every take preserve 2-3 seconds of tail decay?
5. Capture format: 44.1 kHz or 48 kHz, 16-bit or 24-bit?
6. Channel count: mono or stereo?
7. File format: WAV only or another archival format?
8. Use the same qin QIN_A for all takes?
9. Fix tuning to ZHENG_DIAO for the entire session?
10. Allow multiple takes per task?
11. Retake naming: suffix by retake number, quality, or both?
12. Recording order: atomic first then context, or grouped by the existing batches?
13. Start with manual splitting rather than automatic slicing?
14. Record a room-noise sample?
15. Record tuning reference tones before and after the session?
16. Should context takes be promoted into sample_assets immediately or kept as training/reference evidence?
17. Which sample set should render_audio prefer after real samples exist?
